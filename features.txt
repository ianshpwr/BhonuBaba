# BhonuBaba - Premium E-Commerce Platform

BhonuBaba is a full-stack, premium e-commerce platform built with a highly aesthetic dark-peach theme. This document outlines the complete technical context, architecture, and feature set of the application.

## рџ›  Tech Stack
- **Frontend**: Next.js (App Router), TypeScript, Tailwind CSS v4, Lucide React (Icons), React Hot Toast (Notifications).
- **Backend**: Node.js, Express.js (TypeScript), MongoDB Atlas (Mongoose), JSON Web Tokens (JWT), bcryptjs.
- **State Management**: React Context API (`AuthContext` and `CartContext`) mapped to `localStorage` for session persistence.

## рџЋЁ UI & Design System (Dark Peach Theme)
- **Background**: Deep dark (`#0f0f0f`), Cards (`#1f1f1f`).
- **Accent Theme**: Peach gradient (`linear-gradient(135deg, #ff9a8b, #ff6a88)`), Text (`#ffffff` & `#b3b3b3`).
- **Styling**: Extensive use of Glassmorphism (`backdrop-blur(12px)`), soft floating shadows (`.bhonu-shadow`), subtle borders (`#2a2a2a`), completely rounded corners (`rounded-2xl`).
- **Animations**: Global `transition-all duration-300` classes, `hover:scale` micro-animations on cards and buttons, fade-ins on page loads.

## рџљЂ Core Features & Implementation

### 1. Authentication System
- **Backend**: `/api/auth/register` and `/api/auth/login`. Passwords hashed with `bcryptjs`. JWT created and returned to the client. Pre-save Mongoose hook handles password hashing natively.
- **Frontend Context**: `AuthContext.tsx` stores the user securely in `localStorage`. Automatically handles `login()` and `logout()` routines.
- **Protected Routes**: A `<ProtectedRoute>` Next.js wrapper component guards private checkout and order history features by checking Context/localStorage and redirecting to `/auth/login` if unauthenticated.

### 2. Global Cart Ecosystem
- **Context API**: `CartContext.tsx` handles `addToCart`, `removeFromCart`, `updateQuantity`, `clearCart`.
- **Persistence**: Auto-syncs array state into `localStorage` (`bhonubaba_cart`) to survive browser refreshes.
- **Dynamic Math**: Calculates subtotal sums and total badge counts automatically natively inside the Provider.

### 3. Product & Search Architecture
- **Backend API**: `/api/products` and `/api/products/:id` endpoints fetching from MongoDB `Product` schemas.
- **Search System**: The universal Navbar component intercepts queries, applies `useRouter`, and pushes `?search=query` params to the products catalog. The catalog filters natively via `Next.js` `useSearchParams`.

### 4. Order Management
- **Backend**: Protected POST `/api/orders` route to construct an Order document merging the user's ID alongside the mapped Cart items. Fetches historical orders via `GET /api/orders/myorders`.
- **Frontend Views**: Once logged in, Cart can execute Checkout, draining the CartContext, firing a Toast success alert, and routing the user to an elegant historical timeline view at `/orders`.

## рџљЁ Developer Notes & Debugging
- **macOS Port 5000 Conflict**: The backend successfully runs on `PORT 5001` (set in `server/.env`). Port 5000 is intentionally avoided to prevent collisions with the native macOS 12+ AirPlay Control Center Receiver service.
- **Debugging Logs**: Deep `console.log` instrumentation exists in the backend controllers to trace user creation paths securely in the Express terminal without throwing opaque Axios/Fetch errors.
