# PeachStore E-Commerce Features

## 🎨 Theme & UI (Frontend)
- **Aesthetic Peach Theme**: Custom color palette featuring soft peach, pastel pink, and cream hues.
- **Modern UI Elements**: Soft box shadows, rounded corners, and smooth hover micro-interactions.
- **Responsive Design**: Mobile-friendly layout using Tailwind CSS.
- **Custom Components**: Pre-styled reusable `Button`, `Navbar`, `Footer`, and `ProductCard` components.
- **Loading States**: Custom spinning loaders matching the peach aesthetic.

## 🛍️ Frontend Pages
- **Home Page** (`/`): Beautiful hero section and a featured products grid.
- **Product Listing** (`/products`): Displays all available products.
- **Product Details** (`/products/[id]`): Shows detailed product view, price, and descriptions.
- **Shopping Cart** (`/cart`): Interactive cart with subtotal calculation and item removal.
- **Order History** (`/orders`): Lists all past user orders with their delivery status.
- **Authentication** (`/auth/login`, `/auth/register`): Styled forms for user sign-up and login.

## ⚙️ Backend Architecture (Express + Node.js)
- **RESTful API**: Clean `/routes` and `/controllers` modular structure.
- **Database**: Connected to MongoDB Atlas via Mongoose models.
- **Error Handling**: Custom global error handling middleware for cleaner API responses.
- **Security**: Password hashing using `bcryptjs`.
- **Authentication Strategy**: JWT (JSON Web Token) based auth mapping to `req.user`.

## 📦 Database Models
- **User schema**: Name, email, hashed password, and admin roles.
- **Product schema**: Name, descriptions, pricing, dynamic image referencing.
- **Order schema**: Order items array, user references, total pricing, and real-time status tracking.

## 🛠️ Tech Stack Integration
- Built with standard `Next.js` (App Router) on the client side.
- Configured with `ts-node` and `nodemon` for rapid backend development.

## 🐛 Checking Errors (Consoles)
- **Backend Errors**: Check the terminal running `npm run dev` in the `/server` folder. Server crashes, MongoDB connection failures, and API errors are logged here via the custom `errorHandler` middleware.
- **Frontend Errors**: Check the browser's Developer Console (F12 or Inspect -> Console) for client-side issues, failed network requests, or React runtime warnings. Build or compilation errors will appear in the terminal running `npm run dev` inside the `/client` folder.
